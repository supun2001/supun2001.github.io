What Is a SIEM Tool?

A SIEM (Security Information and Event Management) tool is a centralized security platform used to collect, analyse, and correlate logs from across an organisation’s IT environment.

SIEM tools aggregate data from servers, endpoints, network devices, firewalls, applications, and cloud services, then turn that raw data into security alerts, dashboards, and investigations.

In a traditional Security Operations Center (SOC), SIEM tools act as the single source of truth for detecting, investigating, and responding to security incidents.

What Do SIEM Tools Actually Do?

  • Collect logs from multiple systems in real time
  • Normalize and correlate security events
  • Detect suspicious behaviour and known attack patterns
  • Generate alerts for SOC analysts
  • Provide dashboards and reports for visibility and compliance

Common SIEM Tools

  • Splunk Enterprise Security
  • IBM QRadar
  • Microsoft Sentinel
  • Elastic SIEM

While these tools are powerful, they are often expensive to license, complex to maintain, and resource-heavy - especially for students, startups, and small organisations.

Step 1: Deploy the Wazuh Manager

Requirements

  • Ubuntu 24.04 LTS or later version
  • 2 CPU cores
  • 4 GB RAM (minimum)
  • 30 GB disk

Installation

Update the system:

sudo apt update && sudo apt upgrade -y

Install Wazuh using the official installer:

curl -sO https://packages.wazuh.com/4.14/wazuh-install.sh && 
sudo bash ./wazuh-install.sh -a

Learn about more details here

This installs:

  • Wazuh Manager
  • OpenSearch
  • Wazuh Dashboard

After completion, you’ll receive:

  • Dashboard URL
  • Admin username
  • Generated password

Access it via browser:

https://<SERVER-IP>

You now have a working SOC platform.

Step 2: Add Endpoints (Agents)

On any Linux endpoint:

curl -sO https://packages.wazuh.com/4.x/wazuh-agent.sh 
sudo bash wazuh-agent.sh

Configure the agent to point to your Wazuh Manager IP and start it:

sudo systemctl enable wazuh-agent
sudo systemctl start wazuh-agent

Within minutes, logs and security events will appear in the dashboard.

Step 3: Turning Logs into Plain English with a Local LLM (2025 Twist)

The Problem

SOC analysts spend too much time reading cryptic logs like:

Dec 12 sshd[4211]: Failed password for invalid user admin from 185.231.x.x

Beginners struggle to understand:

  • What happened?
  • Is it dangerous?
  • What should I do?

The Solution: Local LLM Log Explanation

Instead of sending logs to cloud AI services, we use a local LLM for:

  • Privacy
  • Cost control
  • Offline analysis

Tooling

  • Ollama
  • Llama (local model)
  • Simple Python integration

Setup Ollama

curl -fsSL https://ollama.com/install.sh | sh
ollama pull llama3

Python Log Explainer (Example)

import requests

def explain_log(log):
    payload = {
        "model": "llama3",
        "prompt": f"Explain this security log in simple SOC terms:\n{log}"
    }
    r = requests.post("http://localhost:11434/api/generate", json=payload)
    return r.json()["response"]

log = "Failed SSH login attempt from external IP"
print(explain_log(log))

What this code actually does:


Wazuh → generates security logs
        ↓
Python script → sends log text
        ↓
Local LLM (Llama) → explains it
“This log indicates a brute-force login attempt over SSH. The source IP is external and may indicate an attacker testing credentials.”

Now your SOC doesn’t just collect logs - it understands them.

Step 4: Monitoring Spam Quarantine Vulnerabilities

Why Spam Quarantine?

Email security issues remain a top attack vector in 2025.

  • Misconfigured spam quarantine systems
  • Attackers bypassing filters
  • Malicious attachments slipping through

What We Monitor in Wazuh

  • Mail server logs (Postfix / Exchange / SMTP gateways)
  • Quarantine release events
  • Suspicious attachment types
  • High-volume spam from single IPs

Custom Wazuh Rule (Example)

<rule id="100500" level="10">
  <if_matched_sid>postfix</if_matched_sid>
  <description>Suspicious spam quarantine activity detected</description>
  <mitre>T1566</mitre>
</rule>

Step 5: Building a Custom Dashboard

In Wazuh Dashboard → Discover / Visualize, create panels for:

  • Quarantined emails per hour
  • Top sender IPs
  • Attachment types (.zip, .html, .iso)
  • Quarantine release actions
“Why were 200 emails released from quarantine in 10 minutes?”

This is how you move from logs → insight → investigation.
